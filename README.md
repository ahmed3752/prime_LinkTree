# Prime Linktree

A beautiful, high-performance, open-source Linktree clone built with the modern Next.js 16 App Router and Supabase. Prime Linktree empowers users to create highly customizable, performant, and tracked "link in bio" pages.

## ✨ Features
- **Core MVP**: Full dashboard for managing links, drag-and-drop reordering (`@dnd-kit`), profile configurations (avatars, bios), and a highly SEO-optimized public profile page.
- **The Customization Engine**: Select from curated themes (`Default`, `Cyber Dark`, `Emerald Glass`) and multiple button styles (`Solid`, `Outline`, `Glassmorphism`) to reflect your brand's aesthetics perfectly.
- **Specialized Links**: Want to organize your links? Drop a **Header** into the mix. Smart links automatically scan for brands (Facebook, Twitter, Github) and inject matching SVG icons seamlessly.
- **Analytics & SEO Tracking**: Real-time traffic interception logging profile views and link clicks directly into an optimized timescale chart on your dashboard. Native `generateMetadata` Next.js configurations ensure flawless Twitter Card and OpenGraph social embeds.

## 🚀 Tech Stack
- **Framework**: Next.js 16 (App Router + React Server Components)
- **Database + Auth**: Supabase (PostgreSQL, Row-Level-Security, PostgREST API)
- **Styling**: Tailwind CSS v4, `tw-animate-css`
- **Component Library**: Shadcn UI (Radix Primitives)
- **Data Visualization**: Recharts

## 💻 Local Setup & Development

### 1. Clone & Install
```bash
git clone https://github.com/yourusername/prime-linktree.git
cd prime-linktree
npm install
```

### 2. Configure Supabase
You will need a Supabase project. Create a `.env.local` file in your root directory and add the following keys from your Supabase Dashboard -> Project Settings -> API:
```env
NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

### 3. Database Schema
Execute the following schema creation scripts in your Supabase SQL Editor:
```sql
-- 1. Profiles Table
CREATE TABLE public.profiles (
  id uuid references auth.users not null primary key,
  username text unique not null,
  full_name text,
  avatar_url text,
  bio text,
  theme jsonb default '{}'::jsonb
);

-- 2. Links Table
CREATE TABLE public.links (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.profiles(id) not null,
  title text not null,
  url text not null,
  subtitle text default '',
  type text default 'link',
  order_index integer not null default 0,
  is_active boolean default true
);

-- 3. Analytics Table
CREATE TABLE public.analytics (
  id bigint generated by default as identity primary key,
  user_id uuid references public.profiles(id) not null,
  event_type text not null check (event_type in ('view', 'click')),
  link_id uuid references public.links(id),
  created_at timestamptz default now() not null
);

-- Note: Ensure you configure Row-Level Security (RLS) policies for all three tables 
-- via your authentication needs in the Supabase Dashboard. 
```

### 4. Run the Development Server
```bash
npm run dev
```
Navigate to `http://localhost:3000` to begin.
